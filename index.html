¿Qué es este sistema?

Objetivo principal
Simular el funcionamiento de una biblioteca real: gestionar libros, registrar usuarios y controlar quién lleva qué libro y cuándo.

Tipo de aplicación
Consola interactiva con menús, colores ANSI y persistencia automática en archivos .json.

📦 Lo que implementamos

  • CRUD completo de Libros y Usuarios
  • Sistema de Préstamos con fechas
  • Historial LIFO con Pila
  • Catálogo con Lista Simple
  • Guardado automático en JSON
  • Validaciones de negocio

public class Prestamo {
  private static final DateTimeFormatter FMT
    = DateTimeFormatter.ofPattern("dd/MM/yyyy");

  // Al REGISTRAR el préstamo
  public Prestamo(int id, Usuario u, Libro l) {
    this.idUsuario     = u.getId();
    this.idLibro       = l.getId();
    this.nombreUsuario = u.getNombre();
    this.tituloLibro   = l.getTitulo();
    this.fechaPrestamo = LocalDate.now()
                          .format(FMT);
    this.fechaDevolucion = ""; // vacío
    this.devuelto = false;
  }

  // Al DEVOLVER el libro
  public void registrarDevolucion() {
    this.devuelto = true;
    this.fechaDevolucion =
      LocalDate.now().format(FMT);
  }
}

estructuras/NodoUsuario.java El Nodo

Un nodo es la "cajita" que vive dentro de la lista enlazada. Guarda exactamente dos cosas.

public class NodoUsuario {

    // 1. El dato que queremos guardar
    public Usuario      dato;

    // 2. Flecha al próximo nodo
    //    null = no hay siguiente
    public NodoUsuario  siguiente;

    public NodoUsuario(Usuario usuario) {
        this.dato      = usuario;
        this.siguiente = null;
    }
}
El NodoUsuario no sabe nada del mundo exterior. La lógica de recorrer vive en ListaSimple.
┌──────────────────┐ │ dato: Usuario 1 │──┐ └──────────────────┘ │ ▼ ┌──────────────────┐ │ dato: Usuario 2 │──┐ └──────────────────┘ │ ▼ ┌──────────────────┐ │ dato: Usuario 3 │──▶ null └──────────────────┘
Los NodoLibro son iguales pero viven dentro de ListaSimple como clase interna, porque solo la lista los necesita.

estructuras/ListaSimple.java La Lista Enlazada

Complejidad:
agregar()O(n) recorre hasta el final
buscar()O(n) nodo por nodo
eliminar()O(n) reconecta punteros
Eliminación — la más delicada:
Antes: [A] → [B] → [C] → null Eliminar B: [A] ──────→ [C] → null A.siguiente apunta directamente a C
public String eliminarLibro(int id) {
    NodoLibro anterior = null;
    NodoLibro actual   = cabezaLibros;

    while (actual != null) {
        if (actual.dato.getId() == id) {

            if (anterior == null)
                // era la cabeza
                cabezaLibros =
                    actual.siguiente;
            else
                // saltamos el nodo
                anterior.siguiente =
                    actual.siguiente;

            tamanoLibros--;
            return "OK: eliminado.";
        }
        anterior = actual;
        actual   = actual.siguiente;
    }
    return "ERROR: no encontrado.";
}

estructuras/Pila.java La Pila — Stack LIFO

LIFO: Último en Entrar → Primero en Salir. El historial siempre muestra del más nuevo al más antiguo.

tope ──▶ ┌─────────────────────┐ │ Préstamo #5 (HOY) │ ← más reciente ├─────────────────────┤ │ Préstamo #4 │ ├─────────────────────┤ │ Préstamo #3 │ ├─────────────────────┤ │ Préstamo #2 │ ├─────────────────────┤ │ Préstamo #1 │ ← más antiguo └─────────────────────┘ Push: O(1) → solo tocamos el tope Peek: O(1) → solo leemos el tope Historial: O(n) → recorremos todo
Nota: usamos la pila como bitácora — no desapilamos. El LIFO se demuestra en el orden de lectura. 
// PUSH: insertar en el tope
public void apilar(Prestamo prestamo) {
    NodoPrestamo nuevo =
        new NodoPrestamo(prestamo);

    // nuevo apunta al ex-tope
    nuevo.siguiente = tope;

    // nuevo PASA a ser el tope
    tope = nuevo;
    tamano++;
}

// PEEK: leer sin extraer
public Prestamo verUltimo() {
    return tope != null
        ? tope.dato
        : null;
}

// Antes: tope→[B]→[A]→null
// Push C: tope→[C]→[B]→[A]→null

// Parseo manual: cuenta llaves { }
private static List
        extraerObjetos(String json) {

    List res = new ArrayList<>();
    int i = json.indexOf("[") + 1;

    while (i < json.length()) {
        if (json.charAt(i) == '{') {
            int prof = 0, j = i;
            while (j < json.length()) {
                char c = json.charAt(j);
                if      (c=='{') prof++;
                else if (c=='}') {
                    prof--;
                    if (prof == 0) {
                        res.add(
                          json.substring(i,j+1));
                        i = j+1; break;
                    }
                }
                j++;
            }
        } else i++;
    }
    return res;
}

servicio/SistemaBiblioteca.java Lógica de negocio

Reglas de negocio:
  • No prestar un libro ya prestado
  • No devolver un libro disponible
  • No eliminar un libro prestado
  • No eliminar usuario con préstamos activos
  • No registrar DNI duplicado
  • Usuario inactivo no puede pedir
public String registrarPrestamo(
        int idUsuario, int idLibro) {

    Usuario u = catalogo
        .buscarUsuarioPorId(idUsuario);
    if (u == null)
      return "ERROR: Usuario no existe.";
    if (!u.isActivo())
      return "ERROR: Usuario inactivo.";

    Libro l = catalogo
        .buscarLibroPorId(idLibro);
    if (l == null)
      return "ERROR: Libro no existe.";
    if (!l.isDisponible())
      return "ERROR: Libro ya prestado.";

    Prestamo p = new Prestamo(
      historial.getSiguienteId(), u, l);

    l.setDisponible(false); // ← clave
    historial.apilarDirecto(p); // PUSH
    guardarTodo();
    return "OK: Préstamo registrado.";
}

    color+"╔"+"═".repeat(n)+"╗"+RESET);

Conclusiones

📚 Lista Simple

Permite recorrer, insertar y eliminar sin arrays. Cada nodo solo conoce al siguiente.

📦 Pila LIFO

El historial respeta último en entrar, primero en salir. Push y Peek en O(1).

🔗 Préstamo

Combina Usuario + Libro y registra fechas automáticamente.

Qué aprendimos

  • Implementar estructuras desde cero con nodos
  • Separación de responsabilidades (capas)
  • Modelar objetos con relaciones
  • Persistir en JSON sin librerías externas

Posibles extensiones

  • Fecha límite y alertas de vencimiento
  • Búsqueda binaria con catálogo ordenado
  • Árbol de búsqueda para acceso más rápido
  • Interfaz web o de escritorio
